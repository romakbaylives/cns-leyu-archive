# CNS Leyu Archive

## 项目简介

**cns-leyu-archive** 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定网站或域名，仅作为静态 HTML 页面的集中存放与版本管理空间。  
每个页面均为独立、自包含的 HTML 文件，可单独查看或部署。

## 目录结构

```
.
├── pages/          # 存放所有归档的 HTML 页面
├── assets/         # 页面引用的公共资源（样式、脚本、图片等）
├── index.html      # 仓库首页（可选，用于导航或目录展示）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或文件即为一个独立的归档页面。
- `assets/` 用于存放页面间可能共享的资源文件，避免重复。
- 页面之间相互独立，无强制依赖关系。

## 页面归档说明

- 所有归档页面均为静态 HTML，不依赖后端服务或数据库。
- 页面内容可能包含文本、图片、样式及脚本，均在本地或通过相对路径引用。
- 每个页面通常包含以下信息：
  - 页面标题
  - 归档日期（可在页面内标注）
  - 原始内容或摘要（如有必要）
- 归档的目的是便于长期保存和离线查看，不保证与原始来源的实时一致性。

## 使用方式

1. **直接查看**：将仓库克隆到本地，使用浏览器打开 `pages/` 下的 HTML 文件。
2. **部署到 GitHub Pages**：将仓库设置为 GitHub Pages 源，即可通过对应 URL 访问 `index.html` 或 `pages/` 下的页面。
3. **自行托管**：将整个仓库内容上传至任何静态文件服务器，即可正常浏览。

## 维护说明

- 本仓库由维护者不定时更新，新增或更新页面会通过提交记录体现。
- 欢迎通过 Issues 提交建议或报告问题，但请注意：
  - 本仓库仅归档静态 HTML 页面，不涉及具体网站管理或内容审核。
  - 不接受涉及侵权、违法或违反平台政策的内容。
- 拉取请求（Pull Request）可用于提交新的归档页面或改进现有内容，但需确保页面内容合法合规。
